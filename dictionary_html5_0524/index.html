HTML(Hyper Text Markup Language)의 정의

해당 웹페이지에서 다른 페이지로 이동할 수 있는 기능(hyper link system)을 가진 문서를 만드는 언어. 즉 웹페이지를 위한 마크업 언어이다.

HTML의 목적

웹사이트의 컨텐츠의 의미 설명 및 구조적 설계(Semantic Markup)

XHTML과 HTML의 차이점

XHTML - EXtensible HTML

HTML과 동등한 마크업 언어로서 HTML에 비해 문법적으로 더 정밀함을 요구하는 언어이다.

HTML과 XHTML의 차이1

XHTML 1.0과 HTML 5의 차이점을 대/소문자 및 태그 닫힘 여부에 따라 비교한 표
구분 XHTML 1.0 HTML 5
태그 소문자만 가능 대/소문자 모두 가능
태그 닫힘 필수 선택(반드시 닫지 않아도 화면에 정상 표출됨)
빈 태그 닫힘 필수( ' ' + / ) 선택
테이블태그 활용 참조

Doctype 선언 방식의 차이

HTML 5와 XHTML 1.0은 doctype을 선언하는 방식에도 차이가 있다.

HTML과 XHTML의 차이2

HTML 5와 XHTML 1.0의 차이점을 Doctype선언 방식에 따라 비교한 표
구분 Doctype
XHTML 1.0 strict <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
HTML 5 <!DOCTYPE html>

HTML 5 태그

기본 틀을 잡는 요소

HTML작성의 기본 구성 요소

HTML문서를 작성할 때 기본적으로 포함되어야 할 태그(요소)를 나타내는 표.
태그 설명
<!DOCTYPE> 문서의 종류를 정의한다.
<html> html 문서를 정의한다.
<head> 해당 문서의 정보에 관련한 데이터(메타데이터)를 나타내는 태그
<title> 문서의 제목을 정의한다.
<body> 문서의 본문을 정의한다.
<h1~h6> html 문서의 제목들을 나타낸다.
<p> 문장을 나타낸다.
<br> 인라인요소 중간에 삽입하여 문장을 한 줄씩 줄바꿈 할 때 사용한다.
<hr> 문단을 의미에 따라 줄바꿈하거나 분리할 때 사용한다.
<!-- ~ --> 주석처리할 때 사용한다.

문장 서식

문장(문단) 서식

HTML문서 작성 중 문장이나 문단의 기본 서식을 지칭하는 태그를 나타내는 표
태그 설명
<abbr> 축약형 영어나 두문자어(고유단어로 발음하는 초성조합 축약어)를 나타낸다.
<address> 문서나 기사의 저자 혹은 주인의 연락처를 나타낸다.
<b> 단순히 html에서 글씨체를 굵게 만들 때 사용.
<em> 특별히 강조하고 싶은 텍스트에 사용하며 기본 기울임꼴로 표현된다.
<strong> 중요한 텍스트에 사용하며 기본 굵은 글자이다.
<i> 단순히 이탤릭체로 기울여서 표현하고 싶을 때 사용. (번외 : fontawesome에서는 i를 이용하여 icon을 표현한다.)
<cite> 미술이나 책, 예술 작품의 제목을 나타내는 태그. 기본 기울임꼴로 표현된다.
<bdi>
Bi-Directional Isolation		 아랍어 등의 일부 언어를 별도로 처리하여 주변 텍스트들과 다른 방향으로 쓰고 싶을 때 사용.
<bdo>
Bi-Directional Override		 하위요소의 텍스트 방향을 정의할 때 사용 (ex .다니입하제이 ~요세하녕안)
<blockquote> 인용된 부분 정의할 때 사용(2줄 이상의 인용구 표현). 들여쓰기로 설정되어있음.
<code> 프로그래밍 코드를 1개씩 명명할 때 사용한다.
<del> 잘못 쓴 문구를 중간밑줄 처리하여 지웠다는 걸 보여주고 싶을때 사용. ins태그와 단짝
<ins> 문서에 추가된 글자를 나타내는 태그. 기본 글자 아래에 줄이 추가되어 표현된다.
<dfn> 해당 문장에서 정의될 용어 자체를 특정하여 나타내고 싶을 때 사용. 기본 기울임꼴로 표현된다.
<kbd> 키보드 입력키를 나타낼 때 사용한다.
<mark> 특정 부분을 형광펜처럼 강조하고 싶을 때 사용. 기본 노란색 바탕으로 강조됨.
<meter> 파일을 업로드할 때 특정 수치의 결과를 볼 때 사용하는 태그. 더 알아보기
<meter id="fuel" min="0" max="100" low="33" high="66" optimum="80" value="50"> at 50/100</meter> at 50/100
<progress> html5에서 추가된 태그로 진행정도를 나타내는 바를 만들 때 사용. 더 알아보기
<progress id="file" max="100" value="70"> 70% </progress> 70%
<pre> 미리 정의된 형식(preformatted)의 텍스트를 나타낼 때 사용한다.
<q> 1줄이하의 인용구를 나타낼 때 사용한다.

입력양식과 입력요소

HTML입력양식과 입력요소

HTML문서를 작성할 때 사용하는 입력양식과 입력요소에 대해 나타낸 표
태그 설명
<form> 입력 양식 전체를 감싸는 태그.
<input> 사용자의 정보를 입력할 수 있는 태그.
<textarea> 2줄 이상의 문자열을 입력할 수 있는 양식 태그.
<button> 링크이동 없이 사용자가 단순 클릭을 할 수 있는 태그.
<select> 선택할 수 있는 목록을 만드는 태그.
<optgroup> 옵션 메뉴를 제공하는 드롭다운 리스트에서 사용되는 옵션들을 그룹으로 묶을 때 사용하는 태그.
<option> 드롭다운 형식으로 선택할 수 있는 요소를 만들 때 사용하는 태그.
<label> 입력요소(input)에 이름을 붙이는 태그.
<fieldset> 입력양식(form)과 관련된 요소들을 묶을 때 사용하는 태그.
<legend> filedset 태그에 이름을 붙일 때 사용.
<datalist> 입력요소에서 사용하기 위한 옵션들의 목록을 미리 정희할 때 사용하는 태그.
<output> 자바스크립트 등으로 실행된 계산의 결과나 사용자의 액션에 의한 결과를 나타낼 때 사용하는 태그.

이미지

이미지

HTML문서 작성 중 이미지와 관련된 모든 태그를 나타내는 표
태그 설명
<img> 이미지를 정의할 때 사용한다.
<map> 이미지에 클릭할 수 있는 영역(이미지맵)을 주고 싶을 때 사용. map은 2개 이상의 area 요서를 포함할 수 있다. 하위로 area요소를 가짐
<area> 이미지맵(클릭할 수 있는 영역을 가진 이미지)에서 영역을 지정할 때 사용하는 태그. 더 알아보기
<canvas> 자바스크립트를 이용하여 그래픽과 애니메이션을 그릴 수 있는 태그.
반드시 닫는 태그가 있어야 한다.html에서 canvas 요소는 단순히 컨테이너 역할을 하며 실제로 벡터를 그리는 동작은 스크립트를 사용하여 구현하는 것이 가장 큰 특징.
<figcaption> figure 요소의 제목을 정의하는 태그. 하위로 figure요소를 가진다.
<figure> 삽화, 다이어그램, 사진 등과 같이 문서의 주요 내용에서 독립적인 컨텐츠를 정의할 때 사용.
문서의 흐름에 영향을 주지 않는 내용을 담아야 한다. figure요소 안에 img 요소가 포함될 수 있다.

이미지 네이밍 규칙 - Img file Naming Convention

이미지 네이밍은 보통 영문소문자, 숫자, 언더스코어를 사용하여 조합하며, 이미지 확장자에 상관없이 순차적으로 용도에 따라 적용한다.
이미지 네이밍 조합은 형태_의미_순서_상태 순으로 조합한다.

img파일 이름을 정하는 규칙

나쁜 예시 좋은 예시 설명

About <meta>

<meta>의 정의
해당 문서의 정보에 관련한 데이터를 나타내는 태그
<meta>의 특징
<head> 태그 내부에 위치하며, HTML 5에서는 <meta>를 통해 웹페이지에서 사용자가 뷰포트를 제어할 수 있도록 하는 속성을 제공

meta태그의 3가지 속성

meta태그의 3가지 속성

메타태그의 3가지 속성 http-equiv, name, content에 대해 정리한 표
속성 속성값 설명
http-equiv 항목명 웹 브라우저가 서버에 명령을 내리는 속성에 대한 항목명을 나타냄
HTML 문서가 응답 헤더와 함께 서버에서 브라우저로 전송되었을 때에만 의미를 지님
content 정보값 meta 정보의 내용 지정
name 정보의 이름 meta 정보의 이름을 지정할 수 있음
지정하지 않을 시 http-equiv와 같은 기능을 함

meta태그 종류

  • 검색엔진 키워드 정의

    <meta name="keyword" content="웹퍼블리셔, 신입 웹퍼블리셔, 이제하, html, css, javascript">

  • 웹페이지에 대한 설명 정의

    <meta name="description" content="markup with html5">

  • 웹페이지 저작자 정의

    <meta name="author" content="leejeha">

  • 모바일에서 뷰포트 너비를 디바이스에 맞추기 위한 정의

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

  • IE 하위버전 호환

    <meta http-equiv="X-UA-Compatible" content="IE=edge">

  • 검색 로봇 제어 참조

    <meta name="Robots" content="noindex, nofollow">

HTML 5에서 변경된 사항

  • HTML 5에서는 <meta>요소의 scheme 속성을 더 이상 지원하지 않으며 문자코딩(character set)을 간단하게 정의 할 수 있는 chrset속성이 추가됨

    HTML 4.01 :<meta http-equiv="content-type" content="text/html; charset=UTF-8">
    HTML 5 :<meta charset="UTF-8">
